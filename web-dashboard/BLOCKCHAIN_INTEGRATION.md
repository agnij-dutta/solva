# Blockchain Integration Guide

This document is for the blockchain developer to coordinate contract deployment and integration with the Solva Web Dashboard.

## Overview

The web dashboard needs specific information from deployed contracts to function properly. This document outlines what's needed and how to provide it.

## Required Information

### 1. Deployment Configuration File

The dashboard expects a `deployments.json` file in the project root (`/Users/agnijdutta/Desktop/solva/deployments.json`).

**Location:** `/Users/agnijdutta/Desktop/solva/deployments.json`

**Format:**
```json
{
  "network": "sepolia",
  "rpc_url": "https://free-rpc.nethermind.io/sepolia-juno/v0_7",
  "contracts": {
    "solvency_verifier": "0x...",
    "solva_token": "0x...",
    "solvency_registry": "0x...",
    "lending_protocol": "0x..."
  },
  "deployed_at": "2024-02-15T12:00:00Z"
}
```

**Note:** This file is already generated by `scripts/deploy.sh`. Just ensure it exists and is up-to-date.

### 2. Contract Addresses

The dashboard specifically needs:

- **solvency_registry** - Used for proof submission and status queries (most critical)
- **solvency_verifier** - Referenced but not directly called by dashboard
- **solva_token** - Optional, for display purposes
- **lending_protocol** - Optional, for display purposes

**How addresses are used:**
- Proof submission: `POST /api/submit-proof` calls the registry
- Status queries: `GET /api/proof-status` reads from the registry
- Event fetching: `GET /api/recent-proofs` queries registry events

### 3. RPC Endpoint

**Current Default:**
```
https://free-rpc.nethermind.io/sepolia-juno/v0_7
```

**For Production:**
- Consider dedicated RPC endpoint (Alchemy, Infura equivalent for Starknet)
- Provide backup endpoints for failover
- Update in `deployments.json` or set via environment variable

**Configuration Priority:**
1. `STARKNET_RPC_URL` environment variable
2. `deployments.json` > `rpc_url` field
3. Default hardcoded URL

## Contract Interface Requirements

### SolvencyRegistry ABI

The dashboard uses a minimal ABI for the registry contract. If you modify the contract interface, update `/Users/agnijdutta/Desktop/solva/web-dashboard/src/lib/starknet.ts`.

**Current Interface:**
```cairo
// Read functions
fn get_solvency_info(issuer: ContractAddress) -> (u64, u256, u256, bool, SolvencyTier)
fn is_solvent(issuer: ContractAddress) -> bool

// Events
event SolvencyVerified {
    issuer: ContractAddress,
    merkle_root: u256,
    total_liabilities: u256,
    tier: SolvencyTier,
    timestamp: u64,
}
```

**If you change the contract:**
1. Update the ABI in `src/lib/starknet.ts`
2. Update type mappings in `mapSolvencyTier()`
3. Update event parsing in `fetchRecentProofs()`

### SolvencyTier Enum

The dashboard expects:
```cairo
enum SolvencyTier {
    None,
    TierC,
    TierB,
    TierA,
}
```

If you add/remove tiers, update the TypeScript enum in `/Users/agnijdutta/Desktop/solva/web-dashboard/src/types/index.ts`:
```typescript
export enum SolvencyTier {
  None = 'None',
  TierC = 'TierC',
  TierB = 'TierB',
  TierA = 'TierA',
}
```

## Deployment Workflow

### Initial Deployment

1. **Deploy Contracts** (you handle this)
```bash
cd /Users/agnijdutta/Desktop/solva
./scripts/deploy.sh --devnet  # or without --devnet for Sepolia
```

2. **Verify deployments.json** was created:
```bash
cat /Users/agnijdutta/Desktop/solva/deployments.json
```

3. **Test Dashboard Integration**
```bash
cd web-dashboard
npm install
npm run dev

# Test config endpoint
curl http://localhost:3000/api/config
```

### Redeployment

If you redeploy contracts:

1. **Deploy new contracts**
2. **Update deployments.json** (automatic if using deploy.sh)
3. **Restart dashboard** to reload config:
```bash
# Development
npm run dev

# Production
pm2 restart solva-dashboard
```

**Note:** The dashboard caches deployment config. Call `resetDeploymentCache()` or restart to reload.

## Common Integration Issues

### Issue: "Failed to load deployment config"

**Cause:** `deployments.json` doesn't exist or is malformed

**Solution:**
```bash
# Check file exists
ls -la /Users/agnijdutta/Desktop/solva/deployments.json

# Validate JSON
cat /Users/agnijdutta/Desktop/solva/deployments.json | jq .

# Redeploy if needed
./scripts/deploy.sh --devnet
```

### Issue: "Failed to query solvency info"

**Cause:** Contract address incorrect or RPC unreachable

**Solution:**
1. Verify contract address in `deployments.json`
2. Test RPC connectivity:
```bash
curl -X POST https://free-rpc.nethermind.io/sepolia-juno/v0_7 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"starknet_chainId","params":[],"id":1}'
```
3. Check contract is deployed:
```bash
sncast call \
  --contract-address <REGISTRY_ADDRESS> \
  --function get_max_proof_age \
  --url https://free-rpc.nethermind.io/sepolia-juno/v0_7
```

### Issue: "Proof submission failed"

**Cause:** Account not configured or insufficient funds

**Solution:**
1. Configure Starknet account:
```bash
sncast account list
sncast account create --name my-account
```
2. Fund account via faucet: https://starknet-faucet.vercel.app/
3. Set account in API request:
```bash
curl -X POST http://localhost:3000/api/submit-proof \
  -H "Content-Type: application/json" \
  -d '{"account": "my-account"}'
```

### Issue: Event parsing errors

**Cause:** Event structure mismatch between contract and dashboard

**Solution:**
1. Check event definition in contract
2. Update parser in `src/lib/starknet.ts` > `fetchRecentProofs()`
3. Test event query directly:
```bash
sncast events \
  --address <REGISTRY_ADDRESS> \
  --from-block latest \
  --to-block latest \
  --url https://free-rpc.nethermind.io/sepolia-juno/v0_7
```

## Contract Upgrade Path

If you upgrade contracts:

### Minor Changes (Same Interface)

1. Deploy new contract
2. Update address in `deployments.json`
3. Restart dashboard

**No code changes needed**

### Major Changes (Interface Modified)

1. Deploy new contract
2. Update ABI in `src/lib/starknet.ts`
3. Update type mappings if needed
4. Update event parsing if needed
5. Test all endpoints
6. Deploy updated dashboard

**Code changes required**

## Monitoring Integration

### Contract Events

The dashboard monitors these events:
- `SolvencyVerified` - Successful proof verification
- `SolvencyFailed` - Failed proof verification (if exists)

**Event Query Range:**
- Fetches last 1000 blocks
- Configurable in `src/lib/starknet.ts`

### RPC Health

Dashboard health depends on:
1. RPC endpoint availability
2. Contract deployment status
3. Event indexing

**Health Check:**
```bash
curl http://localhost:3000/api/health
```
