# Solva Web Dashboard - Implementation Summary

## What Was Built

A complete Next.js backend API layer for the Solva ZK solvency verification protocol, providing HTTP/SSE endpoints that wrap existing CLI scripts and integrate with Starknet.

## Key Features

### 1. Real-Time Proof Generation

- Spawns `prove.sh` script as child process
- Parses stdout to track progress (5 stages)
- Streams updates via Server-Sent Events
- Session-based progress tracking
- Automatic cleanup after 1 hour

**Progress Stages:**
1. Fetching Bitcoin UTXOs (10%)
2. Building Merkle tree (30%)
3. Compiling Noir circuit (50%)
4. Generating witness (70%)
5. Proving with Barretenberg (85%)
6. Local verification (95%)
7. Complete (100%)

### 2. Starknet Integration

- Connects to Starknet via RPC (default: Sepolia)
- Queries SolvencyRegistry contract
- Parses Cairo types to TypeScript
- Fetches and decodes on-chain events
- Error handling for RPC failures

**Contract Interactions:**
- Read: `get_solvency_info()`, `is_solvent()`
- Events: `SolvencyVerified` emission tracking

### 3. Proof Submission

- Wraps `submit_proof.py` script
- Generates calldata with Garaga hints
- Submits via `sncast invoke`
- Returns transaction hash
- Validates proof artifacts exist

### 4. Error Handling

- Custom error classes (ScriptExecutionError, StarknetError, etc.)
- Structured error responses
- HTTP status codes (200, 400, 404, 500)
- Error logging with context

### 5. Configuration Management

- Loads from `deployments.json` (generated by deploy.sh)
- In-memory caching
- Fallback to environment variables
- Validation of required fields

## Integration Points

**Required:**

- Registry must implement expected interface
- Events must follow expected schema
- RPC endpoint must be accessible

**Coordination:**
- ABI defined in `src/lib/starknet.ts`
- Event parsing in `fetchRecentProofs()`
- Type mapping for Cairo enums
